Virtual Stock

An application that helps users who are new to investing to learn about how money could grow, in the style of "virtual gambling". Similar to virtual gambling, our application will not use real money and investment.

Instead, it will allow the user to create investment portfolios, try out buying and selling of stock, and various investment strategies to see how they can grow (or shrink) their money with time.

Features

List of features which are implemented in the Virtual Stock program.

  1. Allow a user to create one or more portfolios with shares of one or more stock.
    • User can create 2 types of portfolios- flexible and inflexible.
    • Inflexible portfolios cannot be edited once created.
    • User can update(buy/sell more stocks) flexible portfolio after creation.
    • Our program supports one user which is active user.
    • User data is stored in file database system.
    • There are 3 ways by which user can create portfolios.

      • Initializing portfolios

      • User can start with a blank portfolio.
      • User can load previous non-empty portfolio from the file database.
      • User can load a single external file that contains list of portfolio in csv format.
        Format for each row (Inflexible) : portfolio name, ticker symbol, quantity, false (flag for inflexible).
        Format for each row (Flexible) : portfolio name, ticker symbol, quantity, true (flag for flexible), transaction date (YYYY-MM-DD), commission fee, buy (true/false)

      All the above options, once selected will overwrite the previous profile in the user database file.

      While loading existing or external portfolios, if there is any IOException because of error during file operations, an appropriate message is displayed.
      An appropriate error message is displayed for invalid inputs. Invalid inputs : Quantity less than 1, Non-integer quantity, commission fee less than or equal to 0.
      If ticker symbol does not exist, then corresponding stock is skipped in the portfolio.

      Adding a new inflexible portfolio

      • User can add a new portfolio into his existing profile from Main Menu option 1 and then selecting inflexible portfolio.
      • User needs to provide input as portfolio name (alphanumeric) & list of stocks to be added to that portfolio.
      • Portfolio name and ticker symbol are case-insensitive.
      • For each stock, user selects a valid ticker symbol and mentions the quantity (positive integer).
      • If there are two stocks of same ticker in the stock list then we combine them into one by updating its quantity to the sum of all of duplicates.
      • Current price for the stock is the latest available price (closing price) for that ticker symbol.
      • Once the portfolio is saved, user cannot update its content.
      • When user adds the portfolio into their profile, it is persisted in the file database, which can be loaded next time they use the program.
      • For invalid inputs, appropriate message is displayed.
      • Invalid inputs : Non-alphanumeric characters for portfolio name, Quantity less than 1, Non-integer quantity, Ticker Symbol not found in the database.

      Adding a new flexible portfolio

      • User can add a new portfolio into his existing profile from Main Menu option 1 and then selecting flexible portfolio.
      • User needs to provide input as portfolio name (alphanumeric) & list of stocks to be added to that portfolio.
      • Portfolio name and ticker symbol are case-insensitive.
      • For each stock, user selects a valid ticker symbol and mentions the quantity (positive integer), valid transaction date (YYYY-MM-DD), commission fee (greater than 0), buy/sell.
      • If there are two stocks of same ticker having the same transaction date and are of same type (buy/sell) in the stock list then we combine them into one by updating its quantity and commission fee to the sum of all of duplicates.
      • Invested price for the stock is the closing price on the transaction date for that ticker symbol.
      • Once the portfolio is saved, user can add/sell more stocks by selecting option 5: to purchase/sell shares in existing portfolio from the main menu.
      • Stocks can only be sold if equivalent amount of stocks are already present in the portfolio.
      • When user adds the portfolio into their profile, it is persisted in the file database, which can be loaded next time they use the program.
      • For invalid inputs, appropriate message is displayed.
      • Invalid inputs : Non-alphanumeric characters for portfolio name, Quantity less than 1, Non-integer quantity, commission fee less than or equal to 0 Ticker Symbol does not exist.

  2. Get list of portfolios.
    • Displays list of flexible and inflexible portfolios (identified by its portfolio name) for the given user.
    • If portfolio list is empty, then corresponding message is displayed.

  3. Examine the composition of a portfolio.
    4. Flexible Portfolio
    • Displays composition of stocks for given portfolio name on a given date.
    • Each composition of stock display ticker symbol of the stock, quantity and percentage of stock quantity in the portfolio.
    • If portfolio name is not present, then corresponding message is displayed.
    Inflexible Portfolio
    • Displays composition of stocks for given portfolio name.
    • Each composition of stock display ticker symbol of the stock, quantity and percentage of stock quantity in the portfolio.
    • If portfolio name is not present, then corresponding message is displayed.

  4. Determine the total value of a portfolio on a certain date.
    • User selects existing portfolio name and a certain date (YYYY-MM-DD).
    • For above inputs, program will provide price of the given portfolio for the given date.
    • Date should be in the past. Date must exist in the stock database.
    • If date format is invalid, then corresponding message is displayed.
    • If portfolio name is not present, then corresponding message is displayed.
    • For flexible portfolio, if stocks are not present in the portfolio (stocks were added after the given date or were sold before the given date) on the given date the value returned will be 0.00
    • For inflexible portfolio, the portfolio value will be displayed for any given date irrespective of when stocks were added to the portfolio.

  5. Cost Basis
    • User can check the cost basis only for flexible portfolios. If the option is selected for an inflexible portfolio then an appropriate error message will be displayed.
    • User can check the cost basis by selecting option 6: to compute cost basis from the main menu.
    • Cost basis (i.e. the total amount of money invested in a portfolio) by a specific date. This would include all the purchases made in that portfolio till that date including commission fee.

  6. Portfolio Performance Over Time

      7. Text Based UI performance chart
    • User can check the Portfolio Performance Over Time only for flexible portfolios. If the option is selected for an inflexible portfolio then an appropriate error message will be displayed.
    • User can check the Portfolio Performance by selecting option 7: to check portfolio performance from the main menu.
    • User can select start date and end date as performance range. The user can enter this range in 3 ways: day, month and year.
    • Performance graph for a portfolio denotes the value of that portfolio over the time range selected by the user
    • The scale of the graph is relative and it is based on the minimum-maximum value.
    • The minimum timestamp of the graph is 5 and the maximum is 30.
    • The scale of the horizontal axis is decided by the range of values of the portfolios, so that there are no more than 50 asterisks on each line.

      GUI performance chart
    • Displays performance of portfolio over a given range using bar chart.
    • Years are displayed over X-axis.
    • Portfolio's value is Y-axis.
    • Particular value of portfolio at given time interval is displayed on its bar.

  7. Persist a portfolio so that it can be saved and loaded (i.e. save to and retrieve from files)
    • Program uses file system to persist the portfolio data created by the user.
    • All the portfolios are stored in a single csv file.
    • Format for each row (Inflexible) : portfolio name, ticker symbol, quantity, flexible (true/false) separated by comma separator
    • Format for each row (Flexible) : portfolio name, ticker symbol, quantity, flexible (true/false), transaction date (YYYY-MM-DD), commission fee, buy (true/false) separated by comma separator
    • Portfolio is persisted for the given user when they save or update(only for flexible) their portfolio.
    • Persisted portfolio can be loaded again as mentioned in Section1 : Initialization of Portfolios.

  8. Two view implementation - Text and GUI
    • Program now supports GUI implementation using Java Swing.
    • Text based and GUI view co-exists.
    • User can select their desired view using appropriate command argument while intializing the application.

      • Text based UI

      • Use args -text for Text based UI.
      • Features supported in text based UI are as follows :
        • 1. Create Flexible/Inflexible portfolios.
        2. View Flexible/Inflexible portfolios.
        3. Check Flexible/Inflexible Portfolio Composition.
        4. View price of Flexible/Inflexible Portfolios.
        5. Buy or sell new stock in flexible portfolio.
        6. Cost basis of Flexible Portfolios
        7. Portfolio Performance of Flexible portfolios.

      GUI using Java Swing

      • Use args -gui for Graphical based UI.
      • Features supported in text based UI are as follows :
        • 1. Create Flexible portfolios.
        2. View Flexible/Inflexible portfolios.
        3. Check Flexible Portfolio Composition.
        4. View price of Flexible Portfolios.
        5. Buy or sell new stocks using quantity on a certain date in Flexible portfolio.
        6. Invest a specific amount in an existing flexible portfolio on a specific date by specifying the weights.
        7. Cost basis of Flexible Portfolios
        8. Dollar Cost Averaging Strategy for Flexible portfolios.
        9. Portfolio Performance of Flexible portfolios.

  9. Dollar Cost Averaging Strategy
    • Invest a fixed amount into an existing portfolio containing multiple stocks, using a specified weight for each stock in the portfolio.
    • "start-to-finish" dollar-cost averaging as a single operation :
      for each strategy, user mentions start and end date (YYYY-MM-DD), frequency of investment (number of days), amount to be invested (>0), commission fee (>=0)
    • User can add multiple valid stocks and specify proportion (%) of money to be invested for each stock.
    • Of course, total percentage (sum of proportion of all stocks within a strategy) will be 100%.
    • Dollar cost averaging supports buying fractional quantity of stock based on the amount to be invested in it.
